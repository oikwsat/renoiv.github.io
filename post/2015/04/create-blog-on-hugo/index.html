背景

今まで何度かブログを作ってきたが，改めて独自ドメインを取って ブログサイトを構築した。

35最定年説よろしく，最近もっぱらマネジメント業務が増えていて コードに触れる機会がめっきり減ってしまっていて危機感を持っていて 強制的にアウトプットする場をつくろうということが主な目的。

ツールの選定基準

  • サーバ運用はしない
  • ランニングコストは掛けたくない
  • ブログ記事を書くために，技術要素が含まれる (vimやshなど)
  • Markdownで書きたい
  • 使っていて楽しい
  • 独自ドメイン
  • アウトプットするモチベーションを維持しやすいしくみ -

試したツール

1. Jekyll + Github Page お試し

まずは，Jekyllを試してみた。 + サイトのDOM構造をゼロから作っていけて，作り始めは楽しい + サイトをリリースするまでに，頑張らなければならない量が多く，心折れてくる
